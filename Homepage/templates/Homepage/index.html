Some helpful tips for writing dynamic HTML

Important Note!!

Always begin your HTML files with

{%verbatim%}{% load static %}{% endverbatim %}

Otherwise you will not load anything.

Adding Images

To include an image in the HTML make sure to add it to the assets directory and then obtain the url relative to the root of that directory.

IMPORTANT!!
Make sure to run this command:

python3 manage.py collectstatic

Every time you add new images or css files or javascript files. If you don't, then the server will not serve the css, javascript or images, and you will be wondering why you don't see your code doing anything.

For example, I have an image located in assets/img/logo.png in order to reference it within this html, I would write the html like the following:

⟨img src='{%verbatim%}{% static "img/logo.png" %}{%endverbatim%}' alt="bison-logo" /⟩

Linking to other pages

Links should are not too difficult, you just have to make sure you are looking exactly where you need to link to inside the urls.py file.

An example link, linking to another internal page would look like this:

⟨a href="{%verbatim%}{% url 'leaderboardIndex' %}{%endverbatim%}"⟩Leaderboard⟨/a⟩

where 'leaderboardIndex' is the name of the link you want to link to, which can be found inside of the urls.py file in the directory of the page you would like to link to. In this case inside the Leaderboard/urls.py

The link looks like this: Leaderboard

Dynamic HTML

If you ever want to access some variable passed into the context, you can access that variable via some code that looks like this:

{%verbatim%}⟨p⟩ Hi, {{ user.name }}. Your email is {{ user.email }}⟨/p⟩{%endverbatim%}

In this case, the context that was passed in would have looked something like this:

def index(request):
 context = {
  "user": {
   "name": "Gabe",
   "email": "myemail@gmail.com"
  }
 }

Which you should be able to find inside of views.py
